# 概要
claude codeとZAPのAPIを使って、Agentに自動でWeb脆弱性診断を実施させる試みです。  
この試みを通じて、以下の検証を行いたいと考えています。
- ブラックボックスのWeb脆弱性診断自動化
- LLMによる自動診断が「本当にビジネスロジックに関する脆弱性を検出できないのか」の確認
- AIパチンコの面白さ　等
  
なお、ymlやmdはclaudeと壁打ちしながら全くコードを書かずに「動けばいいや」の気持ちで作りました。  

# 利用方法
以下の手順でセットアップ後、`docker compose up --build`で起動して、診断が完了するまで待つだけです。  

1. .envファイルに各種情報を記入する  
環境変数に各種設定値を記入します。具体的には以下の3点です。
- anthropic API Key
- ZAP API Key(任意の文字列でOK)
- 診断対象URL
 
2. ZAPのcontextファイルを生成する  
エージェントに生成させるより自分で生成したほうが速いと思ったので、contextファイルは自前で用意する形にしました。  
ターゲットURLや認証情報の設定を行い、exportした.contextファイルをzap-configディレクトリに格納すればOKです。

レポートはmd形式で出力されます。skillにフォーマットを指定すれば、それに沿った形で生成してくれるようです。

# 留意事項
立ち上がるコンテナは3つです。
1. squid  
   claude-codeとzapの通信先を制御するためのプロキシです。  
   通信先を浸炭対象URL及びanthropicのapiのみに制限し、他のドメインに対して診断等を行わないよう制限しています。  
3. zap  
   claude-codeが診断に利用するための脆弱性診断ツールです。  
   ヘッドレスで立ち上がり、AgentはAPI経由でツールを利用します。  
3. claude-code  
   Agentです。zapを利用し、対象URLに対して脆弱性診断を実施、mdでのレポートを生成します。  
  
外部ドメインへのアクセス制御は実施しているつもりですが、おそらくヌケモレがあります。  
また、本検証は自らが作成した環境もしくは許可された環境以外に対して実行しないでください。
